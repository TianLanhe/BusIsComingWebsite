## 用户场景与测试（必填）

### 用户故事 1 - 以更丝滑的首屏轮播理解 App 功能（优先级：P1）

香港城巴用户进入首页后，希望首屏右侧的 App 截图展示像一个现代产品预览，而不是大图下方挂一排小缩略图。用户应能在不被复杂控件打断的情况下快速理解 App 的 4 个核心功能，并能用左右滑动查看相邻功能场景；同一功能场景内的多张截图则像一叠低旋转扑克牌一样展示，点击露出的后方图片后切到最上层。

**优先级原因**：首屏轮播是用户判断产品是否真实、精致、可信的关键区域。当前底部小图堆叠容易显得粗糙，也与用户期待不符。

**独立测试**：在桌面和手机视口打开首页，观察轮播 10 秒并手动左右滑动一次；确认自动切换间隔为约 3 秒，不显示编号、不显示底部缩略图堆叠、不显示常驻左右箭头，且手动滑动只切换功能场景。对多图场景，点击后方露出的牌堆图片后，该图片应切到主图，当前功能场景不改变。

**验收场景**：

1. **给定** 用户打开首页，**当** 停留 10 秒不操作，**则** 功能轮播至少自动切换 2 次，且每次切换围绕单个主手机截图进行。
2. **给定** 用户查看轮播，**当** 轮播处于任一功能页，**则** 页面不显示 `01`、`02`、`03`、`04` 等编号文案。
3. **给定** 用户左右滑动或拖动轮播区域，**当** 滑动距离达到切换阈值，**则** 只切换到相邻功能场景，不切换同场景截图。
4. **给定** 某个功能页有多张截图，**当** 用户点击牌堆中露出的后方图片，**则** 被点击图片切到主图位置，当前功能场景保持不变。
5. **给定** 用户将鼠标悬停、键盘聚焦或正在拖动/触控轮播，**当** 用户仍在交互，**则** 自动切换暂停，避免打断查看。

### 用户故事 2 - 通过香港语境和自然英文理解网站（优先级：P1）

香港用户浏览网站时，希望繁体中文读起来像面向香港交通场景写成的产品文案，而不是简体中文逐句转换。用户应能从 header、首屏、功能、在线查询、下载、FAQ、footer 和状态提示中看到清楚自然的香港书面语。
英文用户浏览网站时，也应看到自然、克制、符合英语产品页面习惯的表达，而不是中文句式的直译搬运。

**优先级原因**：三语内容是项目核心约束。`zh-Hant` 的语气直接影响香港用户是否信任网站和 App。

**独立测试**：切换到繁体中文，抽查全站用户可见文案，确认关键句不是简体文案的字形转换，且明确使用香港交通产品中更自然的表达。

**验收场景**：

1. **给定** 用户选择繁体中文，**当** 查看首页各主要模块，**则** 文案使用香港实用书面语，清楚、自然、不过度口语化，也不过分官方严肃。
2. **给定** 用户查看交通相关功能描述，**当** 页面提到到站、费用、时间、查询和联系时，**则** 用词应贴近香港交通产品语境。
3. **给定** 用户切换 `zh-Hant`、`zh-Hans` 和 `en`，**当** 查看同一功能区域，**则** 三种语言表达同一产品事实和范围边界，但各自使用自然语气，且英文不是中文句式直译。
4. **给定** 用户期待九巴、港铁、铁路、渡轮或完整出行规划，**当** 查看范围说明或 FAQ，**则** 页面清楚说明当前聚焦 Citybus / 城巴，不暗示覆盖范围外能力。

### 用户故事 3 - 看到真实 App 品牌和正确联系入口（优先级：P2）

访问者需要在导航和页脚看到真实 App 品牌标识，并能通过清楚的“联系我们”入口联系开发者。页面不应继续使用占位邮箱或泛化的“支持我们”表述。

**优先级原因**：真实 logo 和真实邮箱能提升可信度，也避免用户误以为联系入口是捐助或支持渠道。

**独立测试**：打开首页，检查 header、footer 和 favicon 相关展示；确认品牌图标来自 Android App 真实图标主体，导航显示“联系我们”，底部邮箱为 `hezhenyu966@gmail.com`。

**验收场景**：

1. **给定** 用户查看页面 header，**当** 看到品牌区域，**则** 品牌图标使用 Android App 真实图标中的巴士主体，而不是通用线框巴士图标。
2. **给定** 用户查看品牌 logo，**当** logo 显示在 header 或 footer，**则** 不包含 Android launcher 的圆角底板或渐变背景，只显示透明背景上的中间巴士主体。
3. **给定** 用户查看导航，**当** 到达右上角联系入口，**则** `zh-Hant` 显示「聯絡我們」，`zh-Hans` 显示「联系我们」，`en` 显示「Contact Us」。
4. **给定** 用户查看页脚，**当** 点击邮箱链接，**则** 打开面向 `hezhenyu966@gmail.com` 的邮件撰写入口。

### 边界情况

- 当用户开启减少动态效果时，系统应减少或停止自动轮播动画，但仍允许用户手动查看全部功能页。
- 当某个功能页只有一张截图时，系统不得显示相邻截图、更多图片暗示或任何缩略图控件。
- 当同一功能页有多张截图时，系统只能通过点击牌堆中露出的后方图片切换主图；横向滑动或拖动只负责切换功能场景，不得切换同场景截图。
- 后方牌堆图片在桌面端使用约 5 度低旋转，在手机端可收敛到约 4 度；后方图片底部不得在视觉上低于主图底部。
- 后方牌堆图片必须保留可辨识的透明度和足够点击面积，不得淡到看不清或只露出不可操作的一小条。
- 场景点点必须可点击，并直接切换到对应功能场景。
- 当用户在轮播中悬停、聚焦、拖动或触控时，系统必须暂停自动切换；交互结束后可恢复自动切换。
- 当用户使用键盘或读屏访问轮播时，系统必须提供可访问的切换方式，但视觉上不得出现常驻大箭头抢占主画面。
- 当用户切换 `zh-Hant`、`zh-Hans`、`en` 时，当前轮播位置和主要页面状态应保持稳定，不因语言切换回到第一项。
- 当 `zh-Hant` 或 `en` 文案读起来像逐句直译、过度口语或过分官方严肃时，必须进入文案修订，不能只以三语字段存在作为通过依据。
- 当官方或 Android 主项目事实与网站文案冲突时，系统必须以 Android 主项目事实为准，并调整网站文案。
- 当用户期待九巴、港铁、铁路、渡轮、其他非城巴交通或完整路线规划时，页面必须保持当前范围边界，不扩展产品承诺。
- 当 logo 裁切后在小尺寸下不清晰时，系统应调整导出尺寸或安全边距，但不得重新使用 launcher 背景底板或通用线框图标。
- 当后续阶段涉及服务端 HTTP API 变更时，必须另行维护 OpenAPI 3.1 YAML；本功能本身不触发 API 变更。

## 需求（必填）

### 功能需求

- **FR-001**：系统必须将首屏功能展示自动切换间隔设为约 3 秒。
- **FR-002**：系统必须让自动轮播在 4 个核心功能页之间切换：常用路线、路线比较、抵站时间 / 路线详情、出门前监测。
- **FR-003**：系统必须支持用户通过左右滑动或拖动切换功能场景；手机支持触控滑动，桌面支持鼠标或触控板拖动。
- **FR-004**：系统必须在轮播中以一个主手机截图作为视觉焦点。
- **FR-005**：系统必须把同一功能场景内的多张截图展示为低旋转阶梯牌堆：一个主图在最上层，后方最多两张图以低透明、轻缩放、小角度旋转露出，并可点击切换到主图。
- **FR-006**：系统不得在轮播中显示 `01`、`02`、`03`、`04` 等编号文案。
- **FR-007**：系统不得显示底部小图堆叠、右下角胶片条、缩略图按钮组、上下图片堆叠或常驻左右箭头作为主要轮播控制。
- **FR-008**：系统必须在用户悬停、聚焦、拖动或触控轮播时暂停自动切换，并在交互结束后恢复或保持合理的可控状态。
- **FR-009**：系统必须尊重减少动态效果偏好，降低或停止非必要自动动画，同时保留手动查看能力。
- **FR-010**：系统必须在单图功能页隐藏任何“还有更多截图”的视觉暗示或交互控件。
- **FR-025**：系统必须让场景点点成为可点击控件，并直接切换到对应功能场景。
- **FR-026**：系统不得让同场景主图点击、滑动或自动播放改变同场景截图；同场景截图只能通过点击后方牌堆图片切换。
- **FR-027**：系统必须确保后方牌堆图片底部不低于主图底部，桌面旋转约 5 度，手机旋转可收敛到约 4 度。
- **FR-011**：系统必须将导航中的“支持我们 / Support”语义改为“联系我们 / Contact Us”语义，并覆盖三种语言。
- **FR-012**：系统必须将页脚用户可见邮箱和邮件链接更新为 `hezhenyu966@gmail.com`。
- **FR-013**：系统不得在用户可见界面保留 `feedback@busiscoming.local`。
- **FR-014**：系统必须使用 Android App 真实图标中的中间巴士主体作为网站品牌 logo。

- **FR-017**：系统必须把全站用户可见文案纳入三语审校范围，包括普通文案、按钮、状态、错误、图片 alt 和无障碍标签。

### 关键实体（涉及数据时填写）

- **功能轮播页**：首屏用于介绍一个 App 核心能力的展示单元。关键属性包括顺序、三语标题、三语说明、截图组、当前主截图、自动切换状态、手动滑动状态和暂停状态。
- **截图组**：归属于某个功能轮播页的一组真实脱敏 App 截图。关键属性包括默认截图、后方牌堆截图、是否只有单图、点击切换状态、底部基线约束和无缩略图约束。
- **品牌 logo 资产**：网站品牌图标。关键属性包括 Android 源资源、裁切范围、透明背景、导出尺寸、header/footer 使用位置和 favicon 适配状态。
- **联系入口**：导航和页脚中的开发者联系方式。关键属性包括三语标签、说明、显示邮箱、邮件链接和旧占位内容移除状态。
- **本地化文案项**：任一用户可见字符串。关键属性包括 `zh-Hant`、`zh-Hans`、`en`、业务含义、适用界面状态和繁体香港语境审校状态。
- **视觉验收证据**：用于证明本功能符合预期的截图或记录。关键属性包括 viewport、语言、轮播状态、logo 状态、联系入口状态和保存位置。

## 假设

- 用户最终确认采用低旋转阶梯牌堆方向：单一主手机截图、后方最多两张同场景截图露出、桌面约 5 度旋转、手机可收敛、后方图底部不低于主图底部，不显示底部小图堆叠。
- 自动轮播按 4 个功能页切换；手动滑动/拖动和场景点点也只切换功能页；同功能页多张截图只通过点击后方牌堆图片查看，不自动逐张乱跳。
- 桌面端不显示常驻左右箭头，手动切换通过拖动、触控板或可访问控件完成。
- `zh-Hant` 采用香港实用书面语，清楚可信，不过度口语化，也不采用过度政府公文腔。
- `zh-Hans` 和 `en` 与 `zh-Hant` 保持同一产品事实；`zh-Hans` 使用自然简体中文，`en` 使用自然克制的英语产品表达，均不做逐句搬运。
- Android 图标前景资源足以裁出透明背景的中间巴士主体；如导出后小尺寸不清晰，可调整安全边距和尺寸，但不能回退到 launcher 背景。
- 本功能不修改服务端 HTTP API、下载业务语义或在线查询后端能力。
