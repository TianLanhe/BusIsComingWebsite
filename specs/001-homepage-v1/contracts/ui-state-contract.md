# UI 状态契约：第一版网站主页

## 适用范围

本契约定义首页中必须保持一致的交互状态和响应式行为。实现阶段可以选择具体组件拆分方式，但不得改变这些用户可见状态。

## 下载分段按钮

### 状态

| 状态 | 触发 | 展示 | 行为 |
|------|------|------|------|
| `default` | 初始、失焦、鼠标离开 | 一个整体按钮，左侧 Android，右侧 iPhone | 不自动下载 |
| `android-expanded` | hover Android、focus Android、触控选择 Android | 整个按钮扩展为 Android APK 可下载面板 | 若 Android manifest 为 `available`，允许进入下载；否则展示暂不可用原因 |
| `iphone-expanded` | hover iPhone、focus iPhone、触控选择 iPhone | 整个按钮扩展为 iPhone 暂未支持面板 | 不跳转、不下载、不伪装失败 |

### 不变量

- iPhone 状态必须始终不可下载，直到产品事实变更。
- 键盘和触控设备必须获得等价的状态反馈。
- 状态变化不得导致按钮周围布局跳动到遮挡其他内容。

## 首屏轮播

### 展示模式

- 桌面端：手机外框 + 右侧简短说明 + 分页点。
- 手机端：手机截图在上，标题说明和分页点在下。

### 轮播顺序

1. 常用路线快速查询
2. 路线结果比较
3. 多班 ETA / 路线详情
4. 出门前短时通知监测

### 不变量

- 切换语言时保留当前轮播项。
- 每个轮播项必须有标题、说明和素材状态。
- 临时 mock 素材必须保留“实现前替换真实截图”的任务标识。

## 语言切换

### 状态

- `zh-Hant`
- `zh-Hans`
- `en`

### 行为

- 用户未选择语言时优先使用浏览器语言。
- 浏览器语言不匹配时默认 `zh-Hant`。
- 用户选择语言后保持偏好。

### 不变量

- 切换语言不得改变页面滚动位置、当前轮播项或下载平台状态。
- 三语内容必须覆盖同一功能范围，不得某个语言缺少限制提示或平台状态。

## 在线查询静态演示

### 状态

- `static-demo`：固定静态演示状态。

### 行为

- 可展示起点、终点、查询按钮和路线结果形态。
- 不向 Citybus、DATA.GOV.HK 或任何后端发起真实路线查询。
- 必须展示“在线查询功能部分受限，完整功能请下载 App”或等价三语提示。

## 响应式视口

### 基准

- 桌面：1440px 宽，参考 Figma node `4:2`。
- 手机：390px 宽，参考 Figma node `4:183`。

### 验收不变量

- 首屏主 CTA、在线查询次 CTA、语言切换和轮播内容可见可操作。
- 核心文字不重叠、不被不可理解地截断。
- 页面不得出现横向滚动作为查看核心内容的必要条件。
