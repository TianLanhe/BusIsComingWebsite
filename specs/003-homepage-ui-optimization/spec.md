## 用户场景与测试（必填）

### 用户故事 1 - 首屏直接下载 Android APK（优先级：P1）

香港城巴通勤用户打开首页后，希望首屏主按钮就是可理解、可直接执行的 Android APK 下载入口，而不是先跳到下载区或再理解 Android/iPhone 分段控件。

**优先级原因**：下载 App 是网站核心转化动作。首屏下载入口不清楚会直接降低用户完成下载的概率。

**独立测试**：在桌面或手机视口打开首页，点击首屏主按钮后，用户开始下载 Android APK；页面不滚动到下载区，也不显示 Android/iPhone 大平台分段框。

**验收场景**：

1. **给定** 用户首次打开首页，**当** 点击首屏「下载 Android APK」主按钮，**则** 用户开始获得当前 Android APK 文件，页面不发生锚点跳转。
2. **给定** 用户查看首屏下载入口，**当** 下载按钮可见，**则** 用户同时看到 APK 版本、文件大小和低权重的 iPhone 暂未支持状态。
3. **给定** 下载不可用或失败，**当** 用户点击首屏下载按钮，**则** 页面在按钮附近展示清楚的失败或暂不可用提示，而不是表现为成功下载。

---

### 用户故事 2 - 查看自动轮播的核心功能展示（优先级：P1）

潜在用户需要在首屏右侧看到更有质感的真实 App 功能展示。展示模块应自动轮播 4 个核心功能点，帮助用户快速理解常用路线、路线比较、多班 ETA / 路线详情、出门前监控与语音提醒。

**优先级原因**：首屏右侧展示决定用户是否相信这是一个真实可用的 App。自动轮播和真实截图能替代现有静态占位。

**独立测试**：打开首页后不操作页面，确认 4 个核心功能点会自动切换；展示模块没有左右箭头；用户聚焦或点按展示区域时自动轮播暂停。

**验收场景**：

1. **给定** 用户打开桌面首页，**当** 停留在首屏超过一个轮播周期，**则** 功能展示自动从当前核心功能点切换到下一项，并带有平滑的深度换层动画。
2. **给定** 用户将鼠标悬停、键盘聚焦或点按展示模块，**当** 用户正在查看截图或文字，**则** 自动轮播暂停，避免打断用户查看。
3. **给定** 用户查看展示模块，**当** 页面处于桌面视口，**则** 截图位于展示模块左侧，文字说明位于右侧；当页面处于手机视口，**则** 截图位于上方，文字说明位于下方。

---

### 用户故事 3 - 手动切换同一功能点的截图图集（优先级：P2）

用户查看某个核心功能点时，希望先看到主图，也能通过点按露出的堆叠截图查看同一功能点下的其他真实截图，而不是被同一功能点的图片自动轮播打断。

**优先级原因**：图集堆叠是本轮视觉升级的关键差异，能展示更多真实界面，同时保持页面简洁。

**独立测试**：查看任一拥有多张截图的功能点，确认序号 1 截图默认作为主图；点按底层露出的截图后，该截图升为主图；没有左右箭头，也不会自动切换同组截图。

**验收场景**：

1. **给定** 某个核心功能点拥有多张截图，**当** 用户首次看到该功能点，**则** 序号 1 的截图作为主图完整展示，其余截图以堆叠方式露出边缘。
2. **给定** 用户点按堆叠中的底层截图，**当** 切换完成，**则** 被点按的截图升为主图，原主图退回堆叠层。
3. **给定** 某个核心功能点只有一张截图，**当** 用户查看该功能点，**则** 页面只展示单张主图，不显示无意义的堆叠切换目标。

---

### 用户故事 4 - 看到更准确的城巴与香港繁中文案（优先级：P2）

香港用户浏览网站时，需要看到更自然的香港繁体中文和更准确的城巴产品口径，避免误以为网站覆盖所有香港巴士公司、九巴或其他交通工具。

**优先级原因**：本网站必须忠于 Android App 当前能力。文案本地化和范围口径会直接影响用户信任。

**独立测试**：切换到繁体中文后，检查关键首屏、功能、在线查询、下载区和 FAQ 文案，确认繁中不是简中文案的字形转换；页面明确聚焦城巴，不泛称覆盖所有香港巴士。

**验收场景**：

1. **给定** 用户选择繁体中文，**当** 查看首页关键文案，**则** 页面使用香港书面语偏自然的表达，并保持正式、清楚、不过度口语化。
2. **给定** 用户查看产品定位、功能展示和在线查询说明，**当** 页面提到查询范围，**则** 文案明确聚焦城巴 / Citybus。
3. **给定** 用户查看范围说明或 FAQ，**当** 用户期待九巴、港铁、铁路、渡轮或完整出行规划，**则** 页面清楚说明这些能力不在当前范围内。
4. **给定** 用户查看核心功能卡片，**当** 看到价格相关功能，**则** 文案强调可看到多程全程总车费，而不是只强调 HK$ 币种显示。

---

### 用户故事 5 - 从局域网和正式 HTTP 入口访问（优先级：P3）

开发者和验收人员需要能从同一局域网的手机或电脑访问本机前后端服务，以便验证手机端真实体验；正式部署时，访问者应能通过标准 HTTP 入口打开前端页面。

**优先级原因**：本轮 UI 改动必须在真实手机和电脑环境下验证；正式部署入口不能只停留在本机回环地址。

**独立测试**：在同一局域网设备上通过开发机 IP 打开前端页面并访问下载入口；正式部署配置可通过标准 HTTP 入口提供前端页面。

**验收场景**：

1. **给定** 开发者启动本地前后端服务，**当** 同一局域网设备访问开发机 IP，**则** 设备可以打开首页并完成核心页面浏览。
2. **给定** 网站进入正式部署配置，**当** 用户访问标准 HTTP 入口，**则** 用户可以打开前端页面，不需要使用本机回环地址。

### 边界情况

- 当同一功能点只有一张截图时，系统必须隐藏堆叠切换目标，避免误导用户以为还有更多图片。
- 当截图仍包含真实地点、站名、路线号、搜索记录或手机系统无关内容时，系统不得把该截图作为前端展示资产。
- 当价格、时间或 ETA 数值出现在截图中时，系统应保留这些数值，用于展示真实数据形态。
- 当用户切换 `zh-Hant`、`zh-Hans`、`en` 时，当前展示的功能点和主图选择应保持稳定，不能因语言切换重置为第一项。
- 当用户使用减少动态效果的系统偏好时，页面应降低或停止非必要轮播动画，但仍允许用户查看全部核心功能点。
- 当 Android 下载失败时，首屏和下载区都必须显示清楚的不可用或失败状态，不得引导用户去 iPhone 或其他平台。
- 当用户期待九巴、港铁、铁路、渡轮、其他交通工具或完整出行规划时，页面必须明确当前聚焦城巴，不提供这些范围外能力。
- 当手机和电脑视口切换时，主下载入口、在线查询入口、展示模块主图、堆叠图和功能文案必须完整可见、可读、可操作。
- 当前端 UI 需要迭代时，后续阶段必须能从本功能文档找到 Figma 文件、关键节点、交互状态和版本说明。
- 本轮不修改下载 API 路径和下载后端业务逻辑；如果后续需要变更服务端 HTTP API，必须另行更新 OpenAPI 3.1 YAML 契约。

- **UI 可视化**：本轮讨论已产出 browser companion mock：下载入口最终方向、展示模块外层自动轮播和内层手动堆叠模型。实现阶段还必须保存桌面与手机截图证据。

- **外部集成与降级**：本轮不接入新的外部实时服务。真实截图来自本地素材目录，进入前端前必须脱敏；下载失败沿用既有不可用状态；轮播动画需要尊重减少动态效果偏好。

## 需求（必填）

### 功能需求

- **FR-001**：系统必须把首屏主下载入口改为直接下载 Android APK 的主操作，不得先跳转到下载区。
- **FR-002**：系统必须移除首屏 Android/iPhone 大平台分段框，并以低权重方式展示 APK 版本、文件大小和 iPhone 暂未支持状态。
- **FR-003**：系统必须把下载区改为 Android 下载详情卡，展示版本、大小、备用下载入口和 iPhone 暂未支持状态。
- **FR-004**：系统必须在首屏和下载区下载失败时展示原位失败或暂不可用提示。
- **FR-005**：系统不得在普通用户界面展示完整 SHA-256 校验值。
- **FR-006**：系统必须将首屏功能展示组织为 4 个核心功能点，并按固定顺序覆盖常用城巴路线快速查询、路线比较、多班 ETA / 路线详情、出门前监控与语音提醒。
- **FR-007**：系统必须让 4 个核心功能点自动轮播，并提供平滑的深度换层动画。
- **FR-008**：系统不得在核心功能自动轮播中显示左右箭头。
- **FR-009**：系统必须在用户悬停、聚焦或点按展示模块时暂停自动轮播，并在用户离开或结束交互后恢复。
- **FR-010**：系统必须支持同一功能点内多张截图的堆叠展示，序号 1 的截图默认作为主图。
- **FR-011**：系统必须允许用户通过点按堆叠中露出的底层截图手动切换主图。
- **FR-012**：系统不得让同一功能点内的多张截图自动轮播，也不得为同组截图提供左右箭头。
- **FR-013**：系统必须在单图功能点隐藏堆叠切换目标。
- **FR-014**：系统必须使用真实 App 截图的脱敏副本作为前端展示素材，不得直接使用未脱敏原图。
- **FR-015**：系统必须替换或遮蔽截图中的真实地点、站名、路线号、搜索记录和手机系统无关内容。
- **FR-016**：系统必须保留截图中的价格、时间和 ETA 数值。
- **FR-017**：系统必须把页面巴士相关表述收窄为城巴 / Citybus 查询，不得暗示覆盖所有香港巴士公司。
- **FR-018**：系统必须明确说明当前不提供九巴、港铁、铁路、渡轮、其他交通工具查询或完整出行规划。
- **FR-019**：系统必须把核心功能卡片中的价格功能改为强调可看到多程全程总车费，而不是只强调 HK$ 币种显示。
- **FR-020**：系统必须把 `zh-Hant` 作为独立香港文案编写，关键文案不得只是 `zh-Hans` 的字形转换。
- **FR-021**：系统必须继续提供 `zh-Hant`、`zh-Hans` 和 `en` 三语切换，三语内容功能覆盖一致。
- **FR-022**：系统必须在桌面展示模块中采用左图右文，在手机展示模块中采用图上文下。
- **FR-023**：系统必须让真实截图不再套手机边框，但保留足够区分页面背景的视觉边界。
- **FR-024**：系统必须支持同一局域网设备通过开发机 IP 访问本地前后端服务。
- **FR-025**：系统必须支持正式部署时通过标准 HTTP 入口访问前端页面，不要求用户使用本机回环地址。
- **FR-026**：系统不得在本轮修改既有 Android APK 下载 API 路径或下载后端业务语义。

### 关键实体（涉及数据时填写）

- **核心功能展示项**：首页首屏右侧的一个功能主题。关键属性包括顺序、三语标题、三语说明、截图图集、当前主图、自动轮播状态和暂停状态。
- **截图图集**：归属于某个核心功能展示项的一组脱敏截图。关键属性包括图片顺序、是否主图、脱敏状态、替换说明和可展示状态。
- **下载入口状态**：首屏与下载区共享的 Android 下载状态。关键属性包括平台、版本、大小、可下载状态、失败状态和 iPhone 暂未支持说明。
- **本地化文案项**：一个用户可见文案单元。关键属性包括 `zh-Hant`、`zh-Hans`、`en` 三语内容、业务含义和繁中是否为独立香港文案。
- **访问入口**：开发或正式部署时用户访问网站的入口。关键属性包括访问场景、可访问范围、端口语义和失败提示。

## 成功标准（必填）

### 可衡量结果

- **SC-002**：用户在首屏点击一次主按钮即可开始 Android APK 下载，且页面不会滚动到下载区。

- **SC-005**：拥有多张截图的功能点中，用户一次点按露出的堆叠截图即可把该截图切换为主图。
- **SC-006**：单图功能点 100% 不显示可点击的空堆叠目标。
