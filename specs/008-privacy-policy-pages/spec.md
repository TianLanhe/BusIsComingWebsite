# 功能规格：隐私政策页面

**功能分支**：`codex/008-privacy-policy-pages`

**创建日期**：2026-06-30

**状态**：Draft

**输入**：用户描述："新增 `/zh-hant/privacy/`、`/zh-hans/privacy/`、`/en/privacy/` 三个网站隐私政策页面。页面只修改网站，不改 Android App；内容覆盖 BusIsComing Android App 和官方网站；适配未来 Android App 隐私政策跳转；页面可索引、进入 sitemap、footer 提供入口，隐私页不显示语言切换。"

## 用户场景与测试（必填）

### 用户故事 1 - 查看清晰的隐私政策（优先级：P1）

作为 BusIsComing 官网访客或未来从 Android App 跳转过来的用户，我希望打开对应语言的隐私政策页后，能快速理解这份政策同时适用于 Android App 和官方网站，并知道 BusIsComing 会为了哪些功能处理哪些资料。

**优先级原因**：这是新增页面的核心价值，也是未来 Android App 和 Google Play Console 可引用隐私政策 URL 的前提。

**独立测试**：直接访问 `/zh-hant/privacy/`、`/zh-hans/privacy/`、`/en/privacy/`，分别检查首屏、摘要卡、正文结构和语言是否正确。

**验收场景**：

1. **给定** 用户直接打开 `/zh-hant/privacy/`，**当** 页面加载完成，**则** 页面显示 `BusIsComing 私隱政策`，并在首屏说明政策适用于 Android App 及官方网站。
2. **给定** 用户直接打开 `/en/privacy/`，**当** 页面加载完成，**则** 页面显示英文隐私政策，并在首屏看到最后更新日期、联系邮箱和四个摘要重点。
3. **给定** 用户阅读正文，**当** 用户查找资料处理范围，**则** 页面能说明常用路线信息以本机保存为主、网站在线试查会处理必要查询资料、通知监测和语音提醒服务于 ETA 提醒。

---

### 用户故事 2 - 从官网发现隐私政策（优先级：P2）

作为官网访问者，我希望能从当前语言的官网 footer 找到隐私政策入口，而不会被主导航中的隐私入口打断了解功能、在线试查和下载 App 的主路径。

**优先级原因**：隐私政策需要可发现，但不应抢占主页核心导航空间。

**独立测试**：分别在三语首页检查 footer，确认隐私政策链接存在且指向当前语言页面；检查主导航和 FAQ 不新增隐私政策入口。

**验收场景**：

1. **给定** 用户正在浏览繁体首页，**当** 用户查看 footer，**则** 能看到“私隱政策”入口并打开 `/zh-hant/privacy/`。
2. **给定** 用户正在浏览英文首页，**当** 用户查看 footer，**则** 能看到 “Privacy Policy” 入口并打开 `/en/privacy/`。
3. **给定** 用户浏览主导航或 FAQ，**当** 用户查找隐私政策入口，**则** 主导航和 FAQ 不新增隐私政策项，核心导航仍聚焦功能、在线试查、FAQ 和联系。

---

### 用户故事 3 - 让搜索和审核发现隐私政策（优先级：P3）

作为网站维护者，我希望三语隐私政策页能被搜索引擎和审核人员稳定发现，并且每个语言版本都有清晰的 canonical、描述和语言关系。

**优先级原因**：隐私页是网站信任页面，也会成为未来 Android App 与 Google Play Console 的引用目标。

**独立测试**：检查三个隐私页的搜索元信息、canonical、三语互链和 sitemap 公开发现路径。

**验收场景**：

1. **给定** 搜索引擎访问任一隐私政策页，**当** 读取页面元信息，**则** 页面允许索引，canonical 指向自身，并声明三个隐私页之间的语言对应关系。
2. **给定** 搜索引擎读取 sitemap，**当** 解析网站 URL，**则** 能发现三个首页 URL 和三个隐私政策 URL，且首页语言组与隐私政策语言组互不混淆。
3. **给定** 未来需要配置 Android App 隐私政策 URL，**当** 选择默认链接，**则** 可使用 `https://www.busiscoming.com/zh-hant/privacy/` 作为默认繁体入口。

---

### 边界情况

- 当用户直接访问三语隐私页时，页面语言必须由 URL 决定，隐私页本身不显示语言切换控件。
- 当用户从隐私页点击品牌或主导航入口时，链接必须返回当前语言首页或首页对应区域，不能停留在隐私页无效锚点。
- 当用户希望了解“是否收集个人资料”时，页面必须避免“完全不处理任何资料”的误解，明确说明不收集账号身份资料、不做广告追踪，但会为路线查询、位置、ETA 和服务稳定性处理必要资料。
- 当用户查找第三方服务说明时，页面必须明确点名 Citybus、DATA.GOV.HK 和 Google Geocoding API，不得只写成泛泛的第三方服务。
- 当用户在手机和电脑访问隐私页时，首屏摘要和正文必须清晰可读，不能出现文字溢出、重叠或关键入口不可见。
- 当用户期待在本页看到 Android 系统备份、微信、Alipay 或 AlipayHK 说明时，页面不主动展开这些非核心内容，以保持政策简洁。
- 当搜索引擎读取语言关系时，首页 hreflang 组和隐私政策 hreflang 组必须分别成立，不能把 privacy 页面 canonical 到首页。

## 宪法对齐（必填）

- **产品来源**：本功能引用 Android 主项目 `README.md`、`AGENTS.md`、Android Manifest、Google reverse geocoding 设计记录、网站在线查询规格和已确认设计文档 `docs/superpowers/specs/2026-06-30-privacy-policy-pages-design.md`。
- **产品范围**：本功能服务官方网站的信任信息、未来 App 隐私政策跳转、下载前信息透明度和联系开发者入口；不新增路线查询能力、下载能力或 App 行为。
- **排除范围**：不提供完整出行路线规划，不提供地铁、铁路、渡轮、九巴或其他非 Citybus 查询能力；不修改 Android App。
- **前后端边界**：前端负责三语隐私页面、站内入口、页面元信息和可访问内容；后端不新增业务能力；既有网站在线试查日志事实仅作为隐私说明依据。
- **API 文档**：N/A。本功能不新增、修改或移除服务端 HTTP API；既有下载 API 与路线查询 API 继续以共享 OpenAPI 文档为权威来源。
- **服务端 DDD 边界**：N/A。本功能不新增或重构服务端 bounded context、领域层、应用层、基础设施层或接口适配层。
- **服务端稳健性**：N/A。本功能不新增服务端入口、goroutine、后台任务或外部实时服务调用。
- **代码可读性**：后续实现如需调整页面判定、SEO 页面组或当前语言链接生成规则，需要用清晰命名表达页面类型与语言关系；仅对非显而易见的路由和 SEO 关系补充中文注释。
- **三语范围**：新增隐私政策标题、首屏、摘要卡、正文五章、footer 入口、页面元信息和可访问标签均必须覆盖 `zh-Hant`、`zh-Hans`、`en`；繁体使用香港本地“私隱政策”语境，英文使用自然克制的产品政策表达，不从中文机械直译。
- **UI 可视化**：已通过本地可视化 mock 比较信息架构，选择“摘要卡 + 正文分节”；设计记录见 `docs/superpowers/specs/2026-06-30-privacy-policy-pages-design.md`。
- **Figma 设计**：涉及前端 UI。后续计划阶段必须在现有 BusIsComing Website Figma 文件中补充 `Privacy Policy Pages - 008` 页面或等效节点，至少覆盖桌面、手机、首屏摘要、正文分节、footer 入口和隐藏语言切换状态。
- **双端适配**：隐私页必须同时适配手机和电脑；后续验证至少覆盖一个手机 viewport 和一个桌面 viewport，确认首屏、摘要卡、正文和 footer 入口均可读可用。
- **外部集成与降级**：本页面不实时调用 Citybus、DATA.GOV.HK 或 Google Geocoding API，只在文案中说明这些服务的使用场景；因此外部服务不可用时隐私页仍应完整展示。
- **验证与提交**：本 Spec Kit specify 产物完成质量校验后提交；后续实现阶段需验证三语页面、footer 入口、SEO、sitemap、构建产物和双端视觉结果。

## 需求（必填）

### 功能需求

- **FR-001**：系统必须提供 `/zh-hant/privacy/`、`/zh-hans/privacy/`、`/en/privacy/` 三个可直接访问的隐私政策页面。
- **FR-002**：每个隐私政策页面必须在首屏明确说明政策适用于 BusIsComing Android App 和 busiscoming.com 官方网站。
- **FR-003**：每个隐私政策页面必须显示最后更新日期和联系邮箱 `hezhenyu966@gmail.com`。
- **FR-004**：每个隐私政策页面必须展示四个摘要重点：不设账号身份资料、不做广告追踪或出售数据、路线资料以本机保存为主、外部服务仅按功能使用。
- **FR-005**：隐私政策正文必须控制为五个章节：适用范围、我们如何处理资料、外部服务、保存使用和安全、你的选择和联系我们。
- **FR-006**：正文必须说明 Android App 的常用路线和起终点信息以设备本机保存为主，但不得做数据库字段级枚举。
- **FR-007**：正文必须说明网站在线试查会处理用户选择或输入的起点、目的地和必要查询资料，并会产生用于排错和服务稳定性的短期服务日志；用户可见文案不得写死具体保留天数。
- **FR-008**：正文必须明确点名 Citybus、DATA.GOV.HK 和 Google Geocoding API，并说明使用当前位置或地址解析时 GPS 坐标可能发送给 Google Geocoding API。
- **FR-009**：正文必须简短说明通知监测和语音提醒用于 ETA 或路线提醒，但不得展开非必要的系统 TTS 技术细节。
- **FR-010**：正文必须说明用户可通过删除 App 内保存路线、撤回系统权限、停止通知监测、清除浏览器网站数据和联系开发者来管理相关资料或偏好。
- **FR-011**：隐私页不得主动提及 Android 系统备份、设备迁移、微信、Alipay 或 AlipayHK package 检查或跳转。
- **FR-012**：隐私页不得显示语言切换控件，页面语言必须由 URL 决定。
- **FR-013**：首页 footer 必须提供当前语言的隐私政策入口；主导航和 FAQ 不得新增隐私政策入口。
- **FR-014**：隐私页中的品牌入口和导航入口必须返回当前语言首页或首页对应区域，不能指向隐私页内不存在的锚点。
- **FR-015**：每个隐私政策页面必须有独立标题、描述、canonical、Open Graph/Twitter 摘要和允许索引的搜索声明。
- **FR-016**：隐私政策页面的 hreflang 必须只在三语隐私页之间互相指向，`x-default` 指向 `/zh-hant/privacy/`。
- **FR-017**：sitemap 必须包含三语首页 URL 和三语隐私政策 URL，并分别维护首页和隐私政策页面的语言对应关系。
- **FR-018**：本功能不得修改 Android App 行为、权限、语言切换或隐私政策跳转逻辑。
- **FR-019**：隐私页必须在手机和电脑上保持正式、清晰、克制的政策页面风格，不得呈现为营销页或冗长法律文本。

### 关键实体（涉及数据时填写）

- **隐私政策页面**：面向用户和审核人员的三语政策页面，关键属性包括语言、路径、标题、最后更新日期、首屏说明、摘要卡、正文章节和联系邮箱。
- **摘要卡**：首屏四个重点承诺或事实，帮助用户快速理解账号身份资料、广告追踪、本机保存和外部服务使用范围。
- **政策章节**：正文五个固定章节，承载适用范围、资料处理、外部服务、保存使用安全和用户选择。
- **SEO 页面组**：同一页面类型下的三语 URL 关系。首页和隐私政策页面分别形成独立语言组。
- **Footer 隐私入口**：官网当前语言页面到当前语言隐私政策页的发现入口。

## 成功标准（必填）

### 可衡量结果

- **SC-001**：用户直接打开任一三语隐私政策 URL 后，能在 10 秒内判断该页面适用于 Android App 和官方网站。
- **SC-002**：三个隐私政策页面 100% 显示对应语言的标题、摘要卡、正文五章、最后更新日期和联系邮箱。
- **SC-003**：隐私政策正文 100% 包含 Citybus、DATA.GOV.HK、Google Geocoding API、GPS 坐标可能发送给 Google、通知监测、语音提醒、短期服务日志和用户选择说明。
- **SC-004**：三个隐私政策页面 100% 不显示语言切换控件。
- **SC-005**：三语首页 footer 100% 提供当前语言隐私政策入口，且主导航和 FAQ 不新增隐私政策入口。
- **SC-006**：搜索引擎可从公开发现信号找到 3 个首页 URL 和 3 个隐私政策 URL，且每个隐私政策页的 canonical 指向自身。
- **SC-007**：手机和电脑主要 viewport 下，隐私页首屏、摘要卡、正文和 footer 入口无文字重叠、无关键内容截断、无不可操作入口。
- **SC-008**：未来 Android App 或 Google Play Console 可直接使用 `https://www.busiscoming.com/zh-hant/privacy/` 作为默认隐私政策 URL。

## 假设

- Android 主项目当前代码能力与官网可下载 APK 能力保持一致，因此隐私政策按主项目当前能力描述。
- 用户可能从官网、搜索结果、Google Play 审核或未来 Android App 内打开隐私政策页。
- 未来 Android App 会支持语言切换，但本轮不修改 App；App 后续可按自身语言直接打开对应隐私政策 URL。
- 隐私政策内容保持简洁，非必要的系统级备份、第三方支付 App 可见性和数据库字段细节不进入用户可见正文。
- 网站在线试查日志的具体保留天数作为内部事实和实现约束保留，不作为用户可见页面文案。
- 本功能不新增服务端 HTTP API，不改变现有下载、在线试查或部署健康检查能力。
- 后续计划阶段需要补充 Figma 设计引用，并把隐私页纳入桌面和手机视觉验证。
