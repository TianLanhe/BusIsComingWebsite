# 在线巴士路线查询设计

**日期**：2026-06-16
**状态**：用户已确认
**范围**：把网站在线查询区从静态演示升级为真实香港巴士路线查询试用。

## 背景

当前首页已有“在线查询”区域，但第一版只是静态演示。用户希望点击首屏“在线查询”或滚动到查询区后，能够输入起点和终点、从地点候选中选择，并查看真实路线结果。

Android 主项目已经实现 Citybus 地点检索、点到点路线查询、站点预览和首程 ETA。Web 版本沿用这些产品事实，但范围收窄为试用查询：不保存路线、不监控、不展示多班 ETA、不提供排序 UI、不展开路线详情。

用户提供的结果卡参考图确认为视觉信息层级来源：大号路线号、上车站到下车站、右侧首程候车状态、底部价格/耗时/步行距离。

## 已确认决策

1. 新建独立 Spec Kit feature：`004-online-bus-query`。
2. 在线查询区采用“左侧说明 + 右侧查询工具”布局，桌面保留现有分栏，移动端上下堆叠。
3. 后端提供 3 个 JSON API：`/api/routes/query_places`、`/api/routes/query_routes`、`/api/routes/query_etas`。
4. API 请求和响应业务字段都放 JSON body；响应统一 envelope：`{ requestId, data, error }`。
5. 地点检索按当前语言调用 Citybus：`zh-Hant -> l=0`、`en -> l=1`、`zh-Hans -> l=2`。
6. 用户只能从地点候选中选择起点和终点；查询接口接收后端签发的 `placeToken`，不接收裸经纬度。
7. 路线接口返回最多 20 条路线，按耗时排序；包含 Citybus P2P 返回的纯巴士换乘路线。
8. ETA 拆成批量接口，前端路线返回后一次性查询全部路线首程 ETA。
9. `placeToken` 有效 15 分钟，`etaToken` 有效 5 分钟，均需要防篡改签名。
10. 地点候选最多 100 条，下拉面板可滚动，可视高度约 8-10 条。
11. 起点和终点默认空；旧静态示例结果删除；支持交换起终点，交换后清空旧结果。
12. 结果卡不显示铃铛、不支持展开详情、不做排序 UI、不做独立刷新按钮。
13. 语言切换会对同一组起终点自动重查路线和 ETA；失败时保留旧结果并轻量提示。
14. 如果用户已改变起点或终点，新查询失败不得展示旧结果。
15. 0 条路线是正常空态，区别于外部服务失败。
16. 前端按结构化数值格式化价格、耗时和步行距离；站名和地点名只显示当前语言一种。
17. 后端输出结构化 stdout 日志；记录起终点名称、经纬度、语言、结果数、耗时、缓存命中、关键错误和关键结果；不记录 Cookie、token、完整外部 URL、第三方原始响应或 HTML；部署层固定保留 7 天。

## 语言验证

已用真实 Citybus endpoint 验证：

- `bsearch_p3.php?l=0` 返回繁体主名称，常带英文副名。
- `bsearch_p3.php?l=1` 返回英文主名称，常带繁体副名。
- `bsearch_p3.php?l=2` 返回简体主名称，常带英文副名。
- `ppsearch_p3.php` 的路线摘要也遵循 `0` 繁体、`1` 英文、`2` 简体。
- `showstops2.php` 对英文有效，但简体站名在已测 606 示例中仍返回繁体；简体站名应通过 DATA.GOV.HK stop API 的 `name_sc` 补齐。

## 备选方案

### 推荐：三接口拆分

地点检索、路线摘要、批量 ETA 分别提供接口。路线结果先展示基础卡片，ETA 作为增强状态合并。这个方案隔离慢速或失败的实时 ETA，状态清晰，符合前后端分离和可观测性要求。

### 同步路线 + ETA

一个路线接口同步返回 ETA。前端简单，但任何 ETA 失败都会拖慢或污染路线结果，不符合已确认的降级规则。

### 三语聚合响应

后端一次查询并返回三语地点和站名。语言切换更快，但外部请求量高、缓存复杂，也不符合“当前语言只展示一种地点/站名”的产品决策。

## UI 设计

桌面端在线查询区保留现有页面节奏：左侧说明本网页提供基础香港巴士路线试用，保存路线、监控和更多详情请下载 App；右侧是查询工具。移动端单列展示，说明在上，工具在下。

右侧工具包含两个地点输入框、交换按钮和查询按钮。输入 1 个字符后 300ms 防抖检索地点；下拉候选滚动展示，用户必须选择候选。结果区初始为空态，点击查询后 loading，路线结果返回后展示纵向卡片列表。卡片信息层级按用户截图：路线号链、上车站到下车站、右侧候车状态、底部价格/耗时/步行距离。

## 后端设计

新增路线查询 bounded context。领域层表达地点候选、路线选项、首程 ETA 查询令牌、查询错误和日志事件；应用层编排地点检索、路线查询、ETA 批量查询、缓存和限流；基础设施层适配 Citybus mobile、DATA.GOV.HK、HMAC、缓存和日志；接口适配层处理 JSON envelope、HTTP 状态码、错误映射和 requestId。

外部集成包括 Citybus `bsearch_p3.php`、`ppsearch_p3.php`、`showstops2.php` 和 DATA.GOV.HK Citybus ETA/stop API。外部服务超时、解析失败、空结果和限流都要被区分记录和映射。

## 错误处理

用户未选择候选、起终点相同、token 过期或篡改时返回可恢复错误。路线查询 0 条是正常空态。外部服务失败或解析失败是可重试错误。ETA 批量失败不清空路线结果，未成功卡片显示“候车暂不可用”。

语言切换后的重查失败保留旧结果；起终点改变后的新查询失败不展示旧路线。
