# 首页 UI 体验优化补充设计

**日期**：2026-06-27  
**状态**：已获用户确认  
**对应分支**：`codex/007-homepage-ui-polish`  
**范围**：在现有首页体验精修基础上，优化桌面首屏截图展示、截图查看交互、手机功能介绍区、路线结果卡片和“车费一眼看清”文案。不新增服务端能力，不修改下载或路线查询 API。

## 原型是验收基准

本设计包含可打开的 HTML 原型：

- [2026-06-27-homepage-ui-polish-followup.html](../prototypes/2026-06-27-homepage-ui-polish-followup.html)

实现和验收必须以该原型、用户提供的三张截图和本文规则共同为基准。原型用于表达比例、层级和信息密度，不要求逐像素照搬。

## 已确认设计决策

### 桌面首屏功能展示区

桌面端右侧 App 功能展示区采用中等放大方案。主截图从当前约 `236px` 宽提升到约 `300px`，截图轨道可扩展到约 `500px`，但不把文字区压到过窄。后排堆叠截图需要和主截图同等比例放大，而不是只放大主图。

不显示独立的放大提示器、悬浮放大图标或说明气泡。主截图本身可点击进入大图模式；可访问名称需要表达“查看截图大图”，但视觉上不额外放提示控件。

### 截图区与文字区手势分离

功能展示区拆成两个明确的交互区域：

- 截图区：拖动或滑动只切换当前功能内的截图，不切换整个功能。
- 文字介绍区：拖动或滑动切换整个功能。
- 场景圆点：继续作为功能场景导航，点击后跳到对应功能。
- 后排堆叠图：点击后只切换当前功能内的主截图。

这条规则优先于旧实现中“整个轮播区域拖动都切功能”的行为。实现时应避免事件冒泡导致用户在截图上拖动时误切到下一项功能。

### 大图查看模式

点击主截图打开大图查看。大图模式支持：

- 放大和缩小查看截图细节。
- 放大后平移查看局部。
- 左右滑动或按钮切换同一功能内的其他截图。
- `Esc`、关闭按钮或点击遮罩关闭。
- 焦点限制在大图对话框内，并提供清楚的无障碍名称。

大图模式中的左右切换只在同一功能截图组内进行，不跨功能，避免用户查看细节时突然跳到另一个功能。

### 手机功能介绍区

手机端功能介绍采用 2 列精简卡片，不加分组胶囊。每个卡片保留图标、标题和一句短说明，但高度、内边距和行间距要尽量压低，让 6 个功能约 3 行展示；未来扩展到 10 个主要功能时约 5 行展示。

桌面端功能介绍布局保持现状不变，不因为手机端压缩而重排桌面端。

### 手机路线结果卡片

手机端路线结果卡片采用紧凑标签值布局：

- 第一行显示路线号和候车状态，路线号降低字号和字重，不再过粗过大。
- 候车状态使用右侧轻量 pill。
- 站点有返回值时使用一行路径展示；站点缺失时折叠为一条“站点资料暂未返回”类提示，不能继续用“上车站 / 下车站”占据大量空白。
- 车费、耗时、步行合并为一行，但每项都必须显示项目名称和值。
- 项目名称和值要有清晰层级差异：项目名称更浅、更小；项目值更深、更重。

桌面端路线结果卡片保持现状不变。

### “车费一眼看清”文案

功能标题统一改为：

- `zh-Hant`：`車費一眼看清`
- `zh-Hans`：`车费一眼看清`
- `en`：`Fare at a glance`

推荐配套说明：

- `zh-Hant`：`比較候選路線時，同頁看清車費、行程時間和步行距離，揀車更直接。`
- `zh-Hans`：`比较候选路线时，同页看清车费、行程时间和步行距离，选择更直接。`
- `en`：`Compare fare, travel time, and walking distance on the same screen before choosing a route.`

不得把用户提出的修改意图原样写入页面，例如“比较城巴方案时，可直接看到多程全程总车费，而不只是币种显示。”这类句子只作为内部需求背景，不作为用户可见文案。

## 响应式规则

桌面端只调整 hero 功能展示区截图组比例和交互；功能介绍区和路线结果卡片桌面布局保持现状。

手机端重点调整功能介绍区和路线结果卡片：

- 功能介绍：2 列卡片，降低 gap、padding 和单卡高度。
- 路线卡片：保持路线号、状态、站点、指标三层结构，整体高度明显低于当前单列布局。
- 长英文和长繁中标题不得溢出按钮或卡片；必要时允许单项内部换行，但整体不能回到高耸单列。

## 验收要点

Vitest 需要覆盖：

- 截图区拖动只切同功能截图，文字区拖动才切功能。
- 点击主截图打开大图，关闭后回到原功能和原截图。
- 大图模式只切换同功能截图，不跨功能。
- 手机功能介绍区存在 2 列紧凑布局的结构信号，桌面布局不变。
- 手机路线卡片中“车费 / 耗时 / 步行”标签和值同时存在，且不再把三项拆成单列。
- “车费一眼看清”三语标题和说明不再包含内部需求措辞。

Playwright 需要覆盖：

- 1440px 桌面：hero 右侧截图组中等放大，后排堆叠图跟随放大，无放大提示器。
- 390px 手机：功能介绍区为 2 列紧凑卡片。
- 390px 手机：路线结果卡片一屏可看到更多候选路线，标签和值层级清楚。
- 大图模式支持打开、缩放、关闭和同功能截图切换。

## 不在本轮范围

- 不重新设计首页品牌、导航整体结构、下载区、FAQ 或 footer。
- 不改变当前 Citybus / 城巴范围，不提供完整出行路线规划，不扩展到港铁、铁路、渡轮或其他交通工具。
- 不新增、修改或移除后端 HTTP API。
- 不新增桌面端功能介绍重排。
- 不在生产页面显示“点击放大”“拖动说明”等可见教学文案或放大提示器。
