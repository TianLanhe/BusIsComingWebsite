# 首页 UI 体验修正设计

## 背景

本设计补充 `007-homepage-ui-polish` 实现后的三项修正：大图查看形态、路线比较与车费文案分工、路线查询结果卡片长站名处理。用户已确认大图查看采用 B 方案，即桌面和手机统一使用图片浮层加极简辅助信息。

## 大图查看

桌面和手机统一采用图片优先的放大浮层，不再呈现带标题栏和多按钮工具栏的 dialog 观感。点击当前主截图后，页面进入暗色遮罩，当前截图以保留手机圆角、阴影和截图比例的方式放大显示。

浮层只保留必要辅助：

- 关闭按钮：右上角一个极简关闭控件。
- 页码：底部或靠近图片底部显示当前同功能截图序号，例如 `1 / 2`。
- 不显示 `Zoom in`、`Zoom out`、`Reset zoom`、上一张、下一张等工具栏按钮。

交互要求：

- 双指缩放或等价触控手势用于放大和缩小。
- 放大后拖动图片查看细节。
- 左右滑动切换同一功能下的截图，不跨功能。
- 点击遮罩、关闭按钮或按 Esc 关闭浮层。
- 关闭后保持原功能和当前截图状态。

## 功能文案

首屏右侧功能展示区第二模块改回“路线比较”主题，不再用“车费一眼看清”作为该模块标题。

建议文案：

- `zh-Hant` 标题：`路線比較更清楚`
- `zh-Hans` 标题：`路线比较更清楚`
- `en` 标题：`Clearer route comparison`
- `zh-Hant` 说明：`同頁查看候選城巴路線的車費、行程時間和步行距離，揀路線前先比較清楚。`
- `zh-Hans` 说明：`同页查看候选城巴路线的车费、行程时间和步行距离，选择路线前先比较清楚。`
- `en` 说明：`Compare candidate Citybus routes by fare, journey time, and walking distance before choosing.`

网页中间 6 个功能介绍的第 6 项继续保留“车费一眼看清”主题，但说明只聚焦车费直接露出，避免和第 2 项“路线比较”重复。

建议文案：

- `zh-Hant` 标题：`車費一眼看清`
- `zh-Hans` 标题：`车费一眼看清`
- `en` 标题：`Fare at a glance`
- `zh-Hant` 说明：`每條候選路線直接顯示車費，毋須點入詳情才知道大約花費。`
- `zh-Hans` 说明：`每条候选路线直接显示车费，不用点进详情才知道大致花费。`
- `en` 说明：`See the fare on each route option without opening details first.`

## 路线结果卡片

路线查询结果卡片中的起点和终点站名在一行内展示。站名过长时截断并显示省略号，不换成两行，避免手机端路线卡高度继续膨胀。

实现约束：

- 起点、箭头、终点保持同一行。
- 起点和终点各自可收缩，超出当前列宽时 `ellipsis` 截断。
- 保留完整文本给浏览器 tooltip 或无障碍文本，避免截断后完全丢失信息。
