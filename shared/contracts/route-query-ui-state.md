# UI 状态契约：在线巴士路线查询

## 范围

本契约约束前端在线查询区的状态机、三语文案、结果展示和失败保留规则。OpenAPI 只定义服务端接口；本文件定义浏览器内交互不变量。

## 输入状态

| 状态 | 起点 | 终点 | 查询按钮 | 结果区 |
|------|------|------|----------|--------|
| 初始 | 空文本，未选择 | 空文本，未选择 | 禁用 | 提示选择起点和终点 |
| 输入中 | 用户自由输入 | 不变 | 禁用，直到选择候选 | 保持当前结果或空态 |
| 候选加载 | 至少 1 字符 | 不变 | 禁用 | 下拉显示 loading |
| 候选可选 | 服务端返回候选 | 不变 | 仍需起终点都已选 | 下拉显示最多 100 条，可视 8-10 条并滚动 |
| 已选择 | 输入框显示候选名称 | 同理 | 两端都选择且不同地点后启用 | 保持当前结果或空态 |
| 字段错误 | 未选择候选或起终点相同 | 未选择候选或起终点相同 | 禁用 | 字段级错误，不发路线请求 |

用户只能从候选中选择地点；自由文本不得触发路线查询。交换起终点后清空旧路线结果，用户需重新查询。

## 查询状态

| 状态 | 触发 | UI 行为 |
|------|------|---------|
| `idle` | 初始或清空 | 结果区显示初始空态 |
| `routeLoading` | 点击查询 | 结果区下半部分 loading；输入仍可编辑 |
| `routeSuccessEtaLoading` | 路线摘要成功 | 展示路线卡；每张卡候车状态为查询中；立即批量请求 ETA |
| `routeSuccessEtaReady` | 批量 ETA 成功 | 合并对应路线卡候车状态 |
| `routeSuccessEtaPartial` | 部分 ETA 不可用 | 成功卡显示等候/即将到站，失败卡显示候车暂不可用 |
| `empty` | 路线查询成功但 0 条 | 显示正常空态，引导尝试附近地点或调整起终点 |
| `routeError` | 新起终点查询失败 | 不展示旧路线，显示可重试错误 |
| `retainedPreviousResult` | 同一组起终点语言切换重查失败 | 保留旧结果，显示“仍显示上次结果”轻量提示 |
| `rateLimited` | 服务端返回限流 | 显示可恢复限流提示 |
| `tokenExpired` | 地点 token 过期 | 提示重新选择地点，清空无法继续使用的选择 token |

## 旧响应处理

- 前端为每次地点检索、路线查询和 ETA 查询生成或传递 `requestId`。
- 组件必须保存当前 query version；任何响应的 requestId 或 query version 不是最新时不得修改当前 UI。
- ETA 响应只能合并到同一次路线查询返回的路线列表中。
- 用户改变起点或终点后，新查询失败不得展示旧路线。
- 用户只切换语言且起终点未变时，重查失败可以保留旧结果并轻量提示。

## 语言切换

- 用户切换 `zh-Hant`、`zh-Hans` 或 `en` 后，已选地点输入框尽量显示当前语言名称。
- 若无法回填当前语言名称，不清空选择，保留旧显示名并继续尝试重查。
- 动态地点和站名只展示当前语言一种，不展示双语副名。
- 错误 UI 只根据 `error.code` 使用前端三语资源翻译，不直接展示后端 `message`。

## 结果卡不变量

- 显示路线号链，例如 `606` 或 `694 → 307`。
- 显示上车站到下车站，例如 `兴华邨兴翠楼 → 渔湾邨`。
- 显示右侧首程候车状态：候车查询中、等候 N 分钟、即将到站或候车暂不可用。
- 显示底部结构化指标：`HK$` 价格、耗时分钟、步行米数。
- 不显示铃铛，不提供监控入口。
- 不提供排序控件、路线详情展开、多班 ETA 或独立刷新按钮。

## 双端布局

- 桌面 1440px：左侧说明 + 右侧查询工具；结果卡纵向列表；下拉不撑开整页。
- 手机 390px：说明在上，查询工具在下；输入、交换按钮、查询按钮、下拉、结果卡和错误态都必须完整可见。
- 下拉可视高度约 8-10 条，超过后内部滚动。
- 按钮、输入框和路线卡必须有稳定尺寸，加载、错误和长站名不得导致布局跳动或文本重叠。
